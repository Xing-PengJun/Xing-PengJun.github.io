学习 .Net Core 笔记，用 .Net Core 进行开发

# First Day（第一天）

1、在项目文件（.csproj）下，<TargetFramework> 是指定当前项目框架

2、<AspNetCoreHostingModel> 是项目当前托管形式

3、InProcess 是进程内托管，即在本地托管如 IIS 托管，进程名称为 “w3wp.exe” 或 “iisexpress.exe”

​ 1）该应用程序托管在 IIS 工作进程中

​ 2）只有一个 Web 服务器

​ 3）性能角度来看，进程内托管优于进程外托管

4、OutOfProcess 是进程外托管，即在 Kestrel 中托管，进程名称为 “dotnet.exe"

​ 1）该应用程序托管在 Kestrel 中

​ 2）Kestrel 可以与反向代理服务器结合使用，例如 IIS，Nginx 或 Apache

​ 3）性能略微低于进程内，因为有内部和外部 Web 服务器之间代理请求的损耗

5、项目中 launchSettings.json 文件是指本地有关信息，如发布不要发布到生产环境当中。

6、信息一般存储在 appsettings.json 文件中。

# The Next Day（第二天）

接第一天从配置文件 appsettings.json 中获取数据

1、构造函数，构造函数生成快捷方式 ctor→Tab

2、Iconfiguration 服务 .NET Core，用于读取各种配置，资源，信息。 （好像是依赖注入信息配置接口）

# Third Day（第三天）

中间断了几天，稍微复习一下第二天

1、两个关于构造函数的快捷键，创建构造函数 **“ctor”，构造函数里面的参 “prop”**

2、声明字段变量带有 readonly 关键字表示不可更改

# Fourth Day（第四天）

.Net Core 中间件（Middleware）

1、Logging（日志记录），StaticFiles（静态文件支持），MVC

2、中间件可同时被访问和请求，中间件是按照添加的顺序执行的

3、C# lambda 表达式（lambda 表达式是一个匿名方法）

# Fifth Day（第五天）

了解委托以及 lambda 表达式

1、委托声明关键字 delegate，委托可以是无参无返也可以是有参有返，或者是有返无参有参无返

//委托声明（有返有参）
public delegate string SayHelloDlg(string Name)

//委托使用
SayHelloDlg Hello=SayHello;
Console.WriteLine(Hello("李"));
Conosle.ReadKey();


//方法
public string SayHello(string Name)
{
	string SayGreet=$"{Name},你好！";
	return SayGreet;
}

2、匿名函数

//匿名函数也是根据委托来的
SayHelloDlg SayHi= delegata (string name)
{
	string rt = $"{Name},这是匿名函数使用";
    return rt;
}

//调用匿名函数

3、lambda 表达式

//匿名委托
SayHelloDlg SayHiTwo=(parameter)=>{method}

//调用lambda表达式
SayHiTwo();

4、事件是一个功能限制的委托变量，声明事件关键字 event

//声明事件

//使用事件，用+=或-=给事件赋值

# Seventh Day（第七天）

在 net core 中安装 MVC

1、将所需的 MVC 服务添加到 net core 依赖注入容器当中

public void ConfigureServices(IServiceCollection services)
{
	services.AddMVC();
}

2、把 MVC 中间件添加到请求处理管道中

app.UseMvcWithDefaulRoute();

3、Net Core 中依赖注入服务

依赖注入特点亮点：1、低耦合。2、提供了高测试性。

//两个服务绑定，后续不进行改变
services.AddSingleton();
services.AddTransient();
services.AddScoped();
